Welcome to ProSync! This guide will help you track your project, provide feedback, and approve work—all in one place.

## 👤 Client Role Overview

As a **Client**, ProSync gives you:

- 🔍 **Transparency**: See exactly what's being worked on

**Your Key Power**: **Milestone Approval**. You decide when work meets your standards!

## 📊 Understanding Your Project

### Project Overview

When you click on a project, you see:

**Header Information**:
- Project title and description
- Your Project Manager
- Target launch date
- Overall progress (e.g., 65%)

**Tabs**:
1. **Overview**: High-level summary
2. **Milestones**: Project phases
3. **Tasks**: Detailed work items
4. **Activity**: Recent updates

### Reading Progress

**Progress Bar**:
```
[████████░░░░] 67%
```

This shows: **67% of milestones approved**

**Calculation**:
```
Progress = (Approved Milestones / Total Milestones) × 100
```

**Example**:
- 4 total milestones
- 2 approved by you
- 1 in review
- 1 not started
- **Progress: 50%** (2/4 approved)

## 🎯 Milestones: Your Key Responsibility

### What Are Milestones?

Milestones are **major phases** of your project. Think of them as **checkpoints** where you review and approve work.

**Typical Milestones**:
1. ✅ Discovery & Wireframes
2. ✅ Design Mockups
3. 🔄 Development (In Progress)
4. ⏳ Testing & Launch (Not Started)

### Milestone Statuses

**What You'll See**:

| Status | Meaning | Your Action |
|--------|---------|-------------|
| **Pending** | Not started yet | No action needed |
| **In Progress** | Team working on it | No action needed |
| **Review** | Ready for you! | **Review & Decide** |
| **Approved** | You approved it | None—it's done! |
| **Rejected** | You requested changes | Monitor revisions |

## ✅ Approving Work

### When Milestone Is Ready

You'll receive a notification:
```
🔔 "Discovery & Wireframes" is ready for your review!
```

### Review Process

1. **Click the Notification** or go to your project
2. **Open the Milestone** in "Review" status
3. **Review Deliverables**:
   - Read the description
   - View uploaded files (if any)
   - Check preview links
   - Test functionality (if applicable)

4. **Make Your Decision**:

#### Option A: Approve ✅

If you're happy with the work:

1. Click **"Approve"** button
2. Confirm approval
3. **Result**:
   - Milestone marked as "Approved"
   - Project progress updates automatically
   - Team notified
   - Next phase can begin

**When to Approve**:
- Work meets your expectations
- All requirements fulfilled
- Quality is good
- Ready to move forward

#### Option B: Request Changes ❌

If work needs revisions:

1. Click **"Request Changes"** button
2. **Write Clear Feedback**:
   ```
   Please make the following changes:
   
   1. Update the homepage hero image to use our new product photo
   2. Change the color scheme to match our brand guide (sent via email)
   3. Add our company tagline to the footer
   
   Let me know if you need any clarification!
   ```
3. Click **"Submit Feedback"**
4. **Result**:
   - Milestone status changes to "Rejected"
   - PM receives your feedback
   - They make revisions
   - You'll review again

**When to Request Changes**:
- Work doesn't match requirements
- Quality issues need fixing
- Missing elements
- Something doesn't feel right

## 💬 Communicating with Your Team

### Adding Comments

You can comment on:
- Projects (general updates)
- Milestones (phase-specific feedback)
- Tasks (detailed work items)

**How to Comment**:
1. Open project/milestone/task
2. Scroll to **Comments** section
3. Write your message
4. Click **"Post Comment"**

**Example Comments**:
```
"Great work on the wireframes! 
I especially like the new navigation structure."

"Quick question: Can we add a contact form to the about page?"

"The logo looks perfect now. Thank you for the quick revision!"
```

### Asking Questions

Don't hesitate to ask if something's unclear:

```
"Hi [PM Name], 

I see the milestone is marked as 'In Progress'. 
Can you give me an estimate of when it'll be ready for review?

Thanks!"
```

- Progress as expected
- Meeting deadlines
- No major issues

- Some delays
- Waiting on approvals
- Minor blockers

## 🚨 Common Scenarios

### Scenario 1: You're Not Happy with the Work

**Don't approve it!** Request changes instead.

**Steps**:
1. Click **"Request Changes"**
2. Write detailed feedback:
   ```
   I appreciate the effort, but this doesn't quite match 
   what we discussed. Specifically:
   
   - The color scheme is too dark (we wanted light/airy)
   - Missing our company mission statement
   - Navigation menu is different from our wireframe
   
   Can we schedule a quick call to align?
   ```
3. Submit feedback
4. Be available for clarification
5. Review revised version

### Scenario 2: You Need to Change Requirements

**Best Approach**:
1. Contact your PM via comment:
   ```
   "We've had a change in direction. Our CEO wants to 
   add a new feature: [describe feature]. 
   
   How does this impact timeline and budget?"
   ```
2. Discuss with PM
3. They'll create new tasks/milestones
4. Agree on adjusted timeline
5. Approve updated plan

### Scenario 3: Project Seems Stuck

**If no updates for a while**:
1. Check the project page
2. Review recent activity
3. Comment:
   ```
   "Hi team, I haven't seen updates in a few days. 
   What's the current status? Any blockers I can help with?"
   ```
4. Your PM will respond with update

### Scenario 4: Emergency Change Needed

**For urgent issues**:
1. Create comment with **URGENT** tag
2. Explain the issue and impact
3. Contact PM directly (phone/email)
4. Work together on solution
5. Adjust plan as needed

## 🎓 Understanding the Full Workflow

### Complete Project Lifecycle (Your View)

```
1. Project Created
   → You receive notification
   → Review project details
   
2. Milestones Planned
   → You see the roadmap
   → Understand the phases
   
3. Team Works on Milestone 1
   → You monitor progress
   → No action needed yet
   
4. Milestone 1 Ready for Review
   → Notification sent to you
   → **Your turn to review!**
   
5. You Approve Milestone 1 ✅
   → Progress: 25% → 33%
   → Team starts Milestone 2
   
6. Repeat for Each Milestone
   → Review → Approve/Request Changes
   → Progress increases
   
7. Final Milestone Approved
   → Project 100% complete!
   → Deliverables transferred
   → Project marked done 🎉
```

## 💡 Pro Tips

### Get the Best Results

1. **Be Responsive**: Quick reviews = faster delivery
2. **Be Specific**: "Change the color" vs "Change hero background to #FF5733"
3. **Be Constructive**: Point out what works AND what needs work
4. **Be Available**: Answer questions quickly
5. **Be Decisive**: Don't overthink—trust your instincts

### Communication Tips

**Good Feedback**:
```
✅ "The homepage looks great! Two small changes:
   1. Make the logo 20% larger
   2. Change button text from 'Learn More' to 'Get Started'
   Everything else is perfect!"
```

**Vague Feedback**:
```
❌ "I don't like it. Can you make it better?"
```
